# CLAUDE.md — Project Context for Claude Code

## Project

bob-hibernate-wake: A secure sleep/wake plugin for Bob (OpenClaw) LLM gateway.
Bash + Python 3, no external dependencies. macOS LaunchAgent-based daemon.

## Key Paths

- `bin/bob-hibernate` — CLI dispatcher (bash)
- `src/listener.py` — Telegram polling daemon (Python 3)
- `scripts/` — Bash helpers (start, stop, wake, sleep, status, doctor)
- `config/config.example.toml` — Config template

## Coding Standards

- No external pip dependencies; stdlib only.
- Never commit secrets (tokens, wake phrases). Use `.env` or env vars.
- Auth by Telegram numeric user/chat IDs only, never usernames.
- Never log raw secrets.
- Gateway binds to loopback only.
- Small, reviewable PRs. Isolate changes to this plugin folder.

## Security Constraints

- Enforce rate limiting and lockouts on wake attempts.
- Validate all Telegram inputs before processing.
- Keep gateway_bind on loopback for local deployments.

## Codex-to-Claude Handoff Workflow

When a plan is generated by Codex (or any planning tool), place it in
`plans/` as a markdown file following the template in `plans/TEMPLATE.md`.

Claude Code should:
1. Read the plan file
2. Validate the plan against this project's security and coding standards
3. Flag any concerns before implementing
4. Implement changes incrementally, committing after each logical unit
5. Run `scripts/doctor.sh` as a sanity check when applicable
